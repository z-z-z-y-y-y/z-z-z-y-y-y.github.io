高斯消元法详解（附洛谷 P3389 AC 代码）

使用场景

高斯消元法是一种经典的数值算法，主要用于求解线性方程组。当方程组规模适中且系数矩阵非奇异时，该方法能够高效地给出精确解。

核心思想

高斯消元法包含两个主要阶段：

  1. 消元阶段：通过初等行变换将增广矩阵转化为上三角矩阵。
  2. 回代阶段：自底向上求解，将矩阵进一步化为单位阵，得到方程组的解。

消元阶段的具体步骤

消元过程需要逐行推进。对于每一行（假设当前为第 i 行），都必须依次执行以下三个关键操作。


    for (int i = 0; i < n; i++)
    {
        ...
    }

### 第一步：选取主元（列主元消去法）

在当前列（第 `i` 列）中，从第 `i` 行开始向下寻找绝对值最大的系数所在行，并将该行交换至第 `i` 行的位置。这一步骤能够有效避免因除数过小导致的数值不稳定问题。

    int max = i;
    for (int j = i + 1; j < n; j++)
    {
        if (fabs(determinant[j][i]) > fabs(determinant[max][i]))
        {
            max = j;
        }
    }
    if (max != i)
    {
        double *temp = determinant[max];
        determinant[max] = determinant[i];
        determinant[i] = temp;
    }

### 第二步：判断主元是否为零

若经过换行处理后，当前的主元（即 `determinant[i][i]`）仍然为零，则说明系数矩阵是奇异的。此时线性方程组可能无解，或者存在无穷多组解。

    if (determinant[i][i] == 0.0)
    {
        printf("No Solution\n");
        return 0;
    }

### 第三步：归一化与消元

-   **归一化**：将当前行的主元化为 `1`。具体做法是将该行所有的元素（包括常数列）都除以当前的主元值。

    for (int j = n; j >= i; j--)
    {
        determinant[i][j] /= determinant[i][i];
    }
    for (int j = i + 1; j < n; j++)
    {
        for (int k = n; k >= i; k--)
        {
            determinant[j][k] -= determinant[i][k] * determinant[j][i];
        }
    }

-   **消元**：利用当前行，将位于该行之下的所有行中对应列（第 `i` 列）的元素消为 `0`。

    for (int i = n - 1; i >= 1; i--)
    {
        for (int j = 0; j < i; j++)
        {
            determinant[j][n] -= determinant[i][n] * determinant[j][i];
        }
    }

完成上述循环后，原始的增广矩阵即被成功转化为上三角矩阵形式。

---

## 回代阶段

完成消元后，矩阵变为上三角形式。此时需要从最后一行开始，自底向上依次消去主元右侧的系数。经过回代，矩阵最终转化为单位阵，此时最后一列的数据即为方程组的解。

    for (int i = n - 1; i >= 1; i--)
    {
        for (int j = 0; j < i; j++)
        {
            determinant[j][n] -= determinant[i][n] * determinant[j][i];
        }
    }

---

## 关键注意事项与易错点提醒

### 1. 关于主元选取的时机
这是本题最容易出错的地方。**绝对不可以**将选取主元与向下消元两个过程分离开来（即先一次性选好每一列的主元，再统一进行消元）。因为在高斯消元的进程中，矩阵的数值是动态变化的。如果在消元前就提前选定了主元，随着后续行变换的进行，最初选定的主元可能已不再是当前列的最优选择，这会直接导致计算结果错误。

**正确做法**：务必在每一行处理开始前，当场实时选取当前列的主元。

### 2. 关于精度处理
在判断主元是否为零时，需要考虑到浮点数运算的精度误差。

-   在本题（洛谷 P3389）的数据范围下，使用 `== 0.0` 的判断方式通常已经足够。
-   但在其他精度要求更高或数据范围更广的场景中，直接判断等于零可能会导致误判。建议改用容差比较的方式，例如：判断绝对值是否小于 `1e-12` 等极小阈值。

#include <stdio.h>
#include <stdlib.h>
#include <math.h>

int main()
{
    int n;
    scanf("%d", &n);
    double **determinant = (double **)malloc(sizeof(double *) * n);
    for (int i = 0; i < n; i++)
    {
        determinant[i] = (double *)malloc(sizeof(double) * (n + 1));
    }
    for (int i = 0; i < n; i++)
    {
        for (int j = 0; j <= n; j++)
        {
            scanf("%lf", &determinant[i][j]);
        }
    }
    for (int i = 0; i < n; i++)
    {
        int max = i;
        for (int j = i + 1; j < n; j++)
        {
            if (fabs(determinant[j][i]) > fabs(determinant[max][i]))
            {
                max = j;
            }
        }
        if (max != i)
        {
            double *temp = determinant[max];
            determinant[max] = determinant[i];
            determinant[i] = temp;
        }
        if (determinant[i][i] == 0.0)
        {
            printf("No Solution\n");
            return 0;
        }   
            for (int j = n; j >= i; j--)
            {
                determinant[i][j] /= determinant[i][i];
            }
            for (int j = i + 1; j < n; j++)
            {
                for (int k = n; k >= i; k--)
                {
                    determinant[j][k] -= determinant[i][k] * determinant[j][i];
                }
            }
    }
    for (int i = n - 1; i >= 1; i--)
    {
        for (int j = 0; j < i; j++)
        {
            determinant[j][n] -= determinant[i][n] * determinant[j][i];
        }
    }
    for (int i = 0; i < n; i++)
    {
        printf("%.2lf\n", determinant[i][n]);
    }
    return 0;
}
