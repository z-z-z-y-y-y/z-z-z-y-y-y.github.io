算法专题————双指针

Posted on 2026-05-16 Edited on 2026-05-17 In 算法学习
对于数组分块类题目的常用算法————双指针

对于类似vector这种可以随机读取的数据类型，如果题目产生了数据分块，那么可以选择使用双指针的算法

核心观点：数据分块 && 数据有序 -> 双指针。这里的指针并不是指在 C 系列的指向内存地址的指针，而是数组的下标。

算法核心流程

1、使数据有序（的sort（）函数，要求数据可以随机读取） 2、使用左右指针，指向数组的头和尾。或者使用快慢指针，两个指针的移动幅度不同。

例题

  • 我会先说明暴力解法要如何操作，再引入双指针算法来降低时间复杂度 ### 283.移动零 #### 暴力解法
  • 本题的暴力解法非常简单，遍历数组，遇到 0 就删除，并在末尾添上 0，如果 0 的数量很多，那么时间复杂度会来到O(n2)

双指针

  • 本题显然是把数组的数据分为了 0 和 非0 两个部分，那么我们就可以使用双指针来完成任务。 我们可以定义两个变量 cur = 0 和 dest = -1，cur 负责遍历数组，dest 表示非0部分的边界，所以我们可以建立一个while循环，终止条件是遍历完成，在此期间，如果cur遇到了非0元素，则dest++ 并与cur指向的数据将换位置，，如此便可以完成任务。 ### 1089.复写0 #### 暴力解法
  • 本题的暴力解法比较难实现，如果仅仅遍历数组，遇到 0 就进行向后复制，那就会导致数据丢失，绝对错误。

双指针

  • 本题的主要难点在于如何不让数据丢失。那么如果从左往右开始遍历的话会导致数据丢失，那么我们不妨从右往左进行遍历。注意我们不能从尾部开始遍历，而是要从开始复制的第一个数据处开始遍历。本质上来看就是先从左往右“预处理”一下，在从右往左正是处理数据。
  • 设定两个指针 cur 和 dest，均指向头，cur 负责遍历数组，dest 负责模拟复写。如果 cur 指向非0数，dest 向前移动 1 位，cur 指向0，dest 向前移动两位。当 dest 到数组尾的时候，遍历停止，此时 cur 指向的就是最后一个复制的位置。但是，要注意的是，假设 dest 超过了数组尾位置，说明此时 cur 应该指向了 0 ，复写后的数组应该是奇数个0。那么我们需要处理这个边界位置，假设不处理，那么结尾肯定是偶数个0，这显然是错误的。 ### 202.快乐数 #### 暴力解法
  • 本题的暴力解法非常简单，就一直循环计算呗，把所有计算结果记录在一个数组中，如果能到1，就是快乐数，如果产生了循环那就不是快乐数。

双指针

  • 从暴力解法中我们不难发现，其实这道题的关键是，产生循环了吗？那这就很容易让人想到一道题环形链表，本质上是很相似的，所以这道题目我们也可以使用快慢指针。如果是快乐数，那环形里就全是1，如果不是快乐数，那么环形里就还有别的数字。那么我们就定义两个指针一个是数本身（slow），一个是运算过一次的数（fast），每次进入循环，slow运算一次，fast运算两次。当这两个数相等的时候就停止运算，如果是1，就是快乐数。这里要说明，为什么一定会相等，可以使用抽屉原理，对于int类型，最大是231 − 1，2.147.483.647是一个10位数，那么进行一次运算后，它会变成多大呢，我们不妨对其放大到9.999.999.999,对于这个数进行一次运算，得到810，也就是说只要我们对一个数进行大于811次运算，则一定可以得到循环的数。 ### 11.盛最多水的容器 #### 暴力解法
  • 本题的暴力解法也非常简单，暴力枚举呗，把每一个情况都计算一遍，就能比较出最大值了，但是这样的时间复杂度是O(n2)，显然要超时的。

双指针

  • 那么我们可以再感性地想一想，究竟什么情况下，一个面积比较大，对于面积公式S = a × b，a 表示宽度就是下标之差，b 表示高度就是数字大小，那么在高度确定的情况下，是不是两个数字离得越远，面积就越大。同理，是不是在宽度去欸的那个的情况下，数字越大，面积越大。那么我们不难想出，如果我们可以从两边向中间不断的收缩，就能找到最大值了。我们定义两个指针 left 和 right，分别指向数组头和尾，在设定一个 max 记录最大面积，然后当$left < right $ ，就进入循环进行比较，如果left比较小，left++，如果right比较小，就right–。这样就能得出结果。

611.有效三角形的个数

暴力解法

  • 本题的暴力解法还是枚举，对于这种数组类题目其实枚举都是能做一点的，不过大多数情况下都会超时

双指针

  • 我们都知道，如果要三个数能组成三角形，那么取两个较小的数字相加，如果可以大于最大的，就可以组成三角形。那题目没说有序，我们不妨先sort一下，让数组变成升序的，那么我们可以设定三个指针，left right cur，left指向数组头，cur指向数组尾，right指向cur前一个，显然left < right < cur，显然left和right指向的数字较小，我们可以让这两个数字相加，与cur指向的数字作比较，如果小，那么left++，如果大，那么已经满足条件了，显然，left在此时的left到right中间任意取值都会大，那么就不用算了，直接+(right - left)，然后cur–，直到cur==2.
